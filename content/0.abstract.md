<!-- Context: Why the need is so pressing or important -->
SPARQL engines continue to diverge in supported versions and language extensions,
with SPARQL 1.2 standardisation coming to an end, and maintenance mode around the corner, the divergence is expected to continue.
<!-- Need: Why something needed to be done at all -->
The increasing heterogeneity increasingly complicates the maintainability of language tools supporting multiple versions and dialects,
and similarly increases the complexities of federated querying.
and comes with increased complexities for language tools supporting multiple versions and dialects.
<!-- Task: What was undertaken to address the need -->
To address this, we introduce Traqula, a modular framework for parsing, generating, and transforming SPARQL queries.
<!-- Object: What the present document does or covers -->
This paper explains Traqula's architecture, its modular design using builder-based inversion of control,
and how these components support modular SPARQL 1.1 and 1.2 parsing and generation.
<!-- Findings: What the work done yielded or revealed -->
Traqula enables round-tripping, supports generic algebra/AST transformations,
and offers substantially faster parsing times than the existing JavaScript based parser.
<!-- Conclusion: What the findings mean for the audience -->
Such a modular approach simplifies maintenance and experimentation while providing reliable support for
heterogeneous query environments.
<!-- Perspectives: What the future holds, beyond this work -->
Looking ahead,
our framework opens the door to support more query languages,
to building generic rewriting tools for SPARQL federation engines, and creating reliable query tooling for heterogeneous environments.
